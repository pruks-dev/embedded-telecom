# การทดลองที่ 3.1 การสร้าง MQTT Subscriber ด้วย Python

## ขั้นตอนการทดลอง

1. ติดตั้ง Paho MQTT Library สำหรับ Python

   ```bash
   pip install paho-mqtt
   ```

2. เขียนโปรแกรม Python เพื่อ Subscribe ข้อมูลจาก Sensor Node

   ```python
   import paho.mqtt.client as mqtt
   import json
   from datetime import datetime

   MQTT_BROKER = "localhost"
   MQTT_TOPIC = "site/+/telemetry"

   def on_message(client, userdata, msg):
       payload = json.loads(msg.payload.decode())
       timestamp = datetime.now().strftime("%H:%M:%S")
       print(f"[{timestamp}] {msg.topic}: {payload}")

   client = mqtt.Client()
   client.on_message = on_message
   client.connect(MQTT_BROKER, 1883, 60)
   client.subscribe(MQTT_TOPIC)
   client.loop_forever()
   ```

3. รันโปรแกรมและเปิด ESP32 ทิ้งไว้
4. สังเกตข้อความที่แสดงทุก 5 วินาที
5. ทดลองเพิ่ม ESP32 ตัวที่สอง (Site B) โดยเปลี่ยน Site ID — สังเกตว่าข้อมูลจากทั้งสอง Site ปรากฏ

# การทดลองที่ 3.2 การสร้าง Dashboard แบบ Real-time

## ขั้นตอนการทดลอง

1. ติดตั้งไลบรารีสำหรับสร้าง Dashboard

   ```bash
   pip install flask flask-socketio
   ```

2. สร้าง Web Application สำหรับแสดงผล

   ```python
   from flask import Flask, render_template
   from flask_socketio import SocketIO, emit
   import paho.mqtt.client as mqtt
   import json

   app = Flask(__name__)
   socketio = SocketIO(app)

   sites = {
       "A-01": {"temperature": "--", "humidity": "--", "light": "--", "status": "offline"},
       "B-01": {"temperature": "--", "humidity": "--", "light": "--", "status": "offline"},
   }

   def on_mqtt_message(client, userdata, msg):
       topic = msg.topic
       payload = json.loads(msg.payload.decode())
       site_id = topic.split("/")[1]

       if "telemetry" in topic:
           sites[site_id].update(payload)
           sites[site_id]["status"] = "online"
       elif "heartbeat" in topic:
           sites[site_id]["status"] = "online"

       socketio.emit("update", sites)

   @app.route("/")
   def index():
       return render_template("dashboard.html", sites=sites)

   if __name__ == "__main__":
       mqtt_client = mqtt.Client()
       mqtt_client.on_message = on_mqtt_message
       mqtt_client.connect("localhost", 1883, 60)
       mqtt_client.subscribe("site/+/telemetry")
       mqtt_client.subscribe("site/+/heartbeat")
       mqtt_client.loop_start()
       socketio.run(app, host="0.0.0.0", port=5000, debug=True)
   ```

3. สร้างไฟล์ `templates/dashboard.html`

   ```html
   <!DOCTYPE html>
   <html>
   <head>
       <title>NOC Dashboard</title>
       <script src="https://cdn.socket.io/4.0.0/socket.io.min.js"></script>
       <script>
           const socket = io();
           socket.on("update", (data) => {
               for (const [id, site] of Object.entries(data)) {
                   document.getElementById(id).innerHTML =
                       `<h2>${id}</h2>
                        <p>Temp: ${site.temperature}°C</p>
                        <p>Hum: ${site.humidity}%</p>
                        <p>Light: ${site.light}</p>
                        <p>Status: ${site.status}</p>`;
               }
           });
       </script>
   </head>
   <body>
       <h1>Telecom NOC Dashboard</h1>
       <div id="A-01">Loading...</div>
       <div id="B-01">Loading...</div>
   </body>
   </html>
   ```

4. เปิดเว็บเบราว์เซอร์ไปที่ `http://localhost:5000`
5. ตรวจสอบการแสดงผลแบบ Real-time — เมื่อค่าจาก ESP32 เปลี่ยน เว็บควรอัปเดตทันที

# การทดลองที่ 3.3 การตรวจจับ Online/Offline Status

## ขั้นตอนการทดลอง

1. เพิ่มกลไกตรวจสอบ Heartbeat Timeout ใน Python Subscriber

   ```python
   import time
   from threading import Thread

   HEARTBEAT_TIMEOUT = 45  # seconds
   last_heartbeat = {}

   def check_heartbeat():
       while True:
           now = time.time()
           for site_id, last_time in last_heartbeat.items():
               if now - last_time > HEARTBEAT_TIMEOUT:
                   sites[site_id]["status"] = "offline"
                   socketio.emit("update", sites)
           time.sleep(5)

   # ใน on_mqtt_message:
   if "heartbeat" in topic:
       last_heartbeat[site_id] = time.time()

   # เริ่ม thread
   Thread(target=check_heartbeat, daemon=True).start()
   ```

2. ทดสอบโดยปิด ESP32 แล้วสังเกต Dashboard — สถานะควรเปลี่ยนเป็น offline ภายใน 45 วินาที
3. เปิด ESP32 อีกครั้ง — สถานะควรกลับเป็น online

| --------- | ---------------- | ---------------------- | ------- |
| ปิด ESP32 A-01 | | | |
| เปิด ESP32 A-01 | | | |

### คำถามท้ายการทดลอง

1. ถ้า Heartbeat Timeout นานเกินไปจะมีผลเสียอย่างไร ถ้าสั้นเกินไปจะมีผลเสียอย่างไร
2. ใน NOC จริงที่มีหลายร้อย Site ควรแสดง Site ที่ offline ก่อนหรือ Site ที่ online ก่อน เพราะเหตุใด
3. นอกจาก Temperature/Humidity/Light มีข้อมูลใดอีกที่ NOC ควรแสดง

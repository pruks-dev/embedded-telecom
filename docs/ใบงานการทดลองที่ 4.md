# การทดลองที่ 4.1 การติดตั้ง MQTT Broker บน Raspberry Pi

## ขั้นตอนการทดลอง

1. ติดตั้ง Raspberry Pi OS (ถ้ายังไม่ได้ติดตั้ง) — ใช้ Raspberry Pi Imager
2. เชื่อมต่อ Pi กับเครือข่าย และ SSH เข้าไป

   ```bash
   ssh pi@raspberrypi.local
   # default password: raspberry
   ```

3. อัปเดตระบบและติดตั้ง Mosquitto MQTT Broker

   ```bash
   sudo apt update && sudo apt upgrade -y
   sudo apt install -y mosquitto mosquitto-clients
   sudo systemctl enable mosquitto
   sudo systemctl start mosquitto
   ```

4. ทดสอบ MQTT Broker

   ```bash
   # Terminal 1 — Subscribe
   mosquitto_sub -h localhost -t "test"

   # Terminal 2 — Publish
   mosquitto_pub -h localhost -t "test" -m "Hello from Pi Gateway"
   ```

5. กำหนดค่าให้ MQTT Broker รองรับการเชื่อมต่อจากภายนอก

   ```bash
   sudo nano /etc/mosquitto/mosquitto.conf
   ```

   เพิ่มบรรทัด:

   ```
   listener 1883 0.0.0.0
   allow_anonymous true
   ```

6. รีสตาร์ท Mosquitto

   ```bash
   sudo systemctl restart mosquitto
   ```

7. ทดสอบเชื่อมต่อจาก ESP32 โดยเปลี่ยน MQTT Server เป็น IP Address ของ Pi

   ```cpp
   const char* mqtt_server = "192.168.x.x"; // IP of Pi
   ```

# การทดลองที่ 4.2 Protocol Translation — ESP32 MQTT → REST API

## ขั้นตอนการทดลอง

1. เขียน Python Service บน Pi ที่รับข้อมูล MQTT จาก ESP32 แล้วแปลงเป็น REST API

   ```python
   from flask import Flask, jsonify
   import paho.mqtt.client as mqtt
   import json
   from threading import Lock

   app = Flask(__name__)
   data_lock = Lock()
   sensor_data = {}

   def on_message(client, userdata, msg):
       global sensor_data
       topic = msg.topic
       payload = json.loads(msg.payload.decode())
       site_id = topic.split("/")[1]

       with data_lock:
           if site_id not in sensor_data:
               sensor_data[site_id] = {}
           sensor_data[site_id].update(payload)
           sensor_data[site_id]["last_update"] = msg.timestamp if hasattr(msg, 'timestamp') else None

   @app.route("/api/sites")
   def get_sites():
       with data_lock:
           return jsonify(sensor_data)

   @app.route("/api/sites/<site_id>")
   def get_site(site_id):
       with data_lock:
           data = sensor_data.get(site_id)
           if data:
               return jsonify(data)
           return jsonify({"error": "Site not found"}), 404

   if __name__ == "__main__":
       client = mqtt.Client()
       client.on_message = on_message
       client.connect("localhost", 1883, 60)
       client.subscribe("site/+/telemetry")
       client.loop_start()
       app.run(host="0.0.0.0", port=5000)
   ```

2. รัน Service และทดสอบจากคอมพิวเตอร์

   ```bash
   curl http://<PI_IP>:5000/api/sites
   curl http://<PI_IP>:5000/api/sites/A-01
   ```

3. ควรเห็น JSON ที่มีข้อมูล Sensor ล่าสุด

# การทดลองที่ 4.3 Data Aggregation บน Edge Gateway

## ขั้นตอนการทดลอง

1. เพิ่มฟังก์ชัน Data Aggregation บน Pi — คำนวณค่าเฉลี่ย Min/Max ก่อนส่งต่อ

   ```python
   from collections import deque
   import time

   # เก็บประวัติข้อมูล 10 ค่าล่าสุด
   history = {
       "A-01": deque(maxlen=10),
       "B-01": deque(maxlen=10),
   }

   def aggregate_site_data(site_id):
       if not history.get(site_id):
           return None
       temps = [d["temperature"] for d in history[site_id] if "temperature" in d]
       hums = [d["humidity"] for d in history[site_id] if "humidity" in d]

       if not temps:
           return None

       return {
           "site_id": site_id,
           "temperature_avg": sum(temps) / len(temps),
           "temperature_min": min(temps),
           "temperature_max": max(temps),
           "humidity_avg": sum(hums) / len(hums),
           "humidity_min": min(hums),
           "humidity_max": max(hums),
           "sample_count": len(temps),
           "period_seconds": 50,  # 10 samples * 5s interval
       }

   @app.route("/api/sites/<site_id>/aggregate")
   def get_aggregate(site_id):
       result = aggregate_site_data(site_id)
       if result:
           return jsonify(result)
       return jsonify({"error": "No data"}), 404
   ```

2. ทดสอบอ่านข้อมูลรวม

   ```bash
   curl http://<PI_IP>:5000/api/sites/A-01/aggregate
   ```

3. ควรได้ค่าเฉลี่ย Min/Max ของอุณหภูมิและความชื้นย้อนหลัง 10 ตัวอย่าง

### คำถามท้ายการทดลอง

1. การทำ Data Aggregation บน Edge Gateway มีข้อดีอย่างไรเมื่อเทียบกับการส่งข้อมูลดิบทั้งหมดไปยัง Cloud
2. ถ้าต้องการเก็บข้อมูล 1 ชั่วโมง ควรกำหนดขนาดของ `deque` อย่างไร (เมื่อส่งข้อมูลทุก 5 วินาที)
3. Protocol Translation มีประโยชน์อย่างไรในระบบที่มีอุปกรณ์หลายประเภท

2. เชื่อมต่อระบบตามสถาปัตยกรรมต่อไปนี้

   ```
   ┌─────────────────────────────────────────────────────┐
   │                   Site A-01 (ESP32)                │
   │  ┌──────────┐  ┌──────────┐  ┌──────────────────┐  │
   │  │ DHT11    │  │ PIR      │  │ OLED Display     │  │
   │  │ LDR      │  │ Obstacle │  │ Buzzer / Relay   │  │
   │  └────┬─────┘  └────┬─────┘  └──────────────────┘  │
   │       │             │                               │
   │       └──────┬──────┘                               │
   │              │  MQTT                                │
   └──────────────┼──────────────────────────────────────┘
                  │
   ┌──────────────┼──────────────────────────────────────┐
   │              │  Edge Gateway (Raspberry Pi)         │
   │              ▼                                      │
   │  ┌─────────────────────┐                            │
   │  │ MQTT Broker         │                            │
   │  │ (Mosquitto)         │                            │
   │  └──┬──────┬──────┬───┘                            │
   │     │      │      │                                │
   │     ▼      ▼      ▼                                │
   │  ┌────┐ ┌────┐ ┌────┐                              │
   │  │NOC │ │LLM │ │n8n │                              │
   │  │Dash│ │AI  │ │Auto│                              │
   │  └────┘ └────┘ └────┘                              │
   │     │      │      │                                │
   └─────┼──────┼──────┼────────────────────────────────┘
         │      │      │
         ▼      ▼      ▼
      Browser  Alert  LINE/Email
                     /Incident
   ```

3. รันบริการทั้งหมดบน Raspberry Pi

   ```bash
   # 1. MQTT Broker (ตรวจสอบว่าทำงานอยู่)
   sudo systemctl status mosquitto

   # 2. NOC Dashboard (Python Flask)
   cd ~/noc-dashboard
   python3 app.py &

   # 3. LLM Analyzer (Python)
   cd ~/edge-ai
   python3 analyzer.py &

   # 4. n8n Workflow
   n8n start &
   ```

4. เปิด ESP32 โดยให้เชื่อมต่อ WiFi และ MQTT Broker (IP ของ Pi)
5. ตรวจสอบว่าระบบทั้ง 4 บริการทำงานพร้อมกัน

# การทดลองที่ 8.2 การทดสอบ End-to-End Scenario

## ขั้นตอนการทดลอง

1. ทดสอบ Scenario ปกติ (Normal Operation)

   | ขั้นตอน | สิ่งที่ควรเกิดขึ้น |
   | ------- | ---------------- |
   | ESP32 ส่ง Telemetry ทุก 5s | ข้อมูลปรากฏบน NOC Dashboard |
   | OLED แสดง SITE A-01 ปกติ | LED เขียวติด |
   | Heartbeat ทุก 30s | Status = Online |

2. ทดสอบ Scenario Alarm

   | ขั้นตอน | สิ่งที่ควรเกิดขึ้น |
   | ------- | ---------------- |
   | เปิด PIR (Motion) | Alarm Level = MAJOR, LED เหลือง/แดงติด, Buzzer ดัง |
   | บัง Obstacle (Door Open) | Alarm ปรากฏบน Dashboard |
   | ทำทั้งสองอย่างพร้อมกัน | Alarm Level = CRITICAL, Relay ทำงาน |

3. ทดสอบ Scenario Automation

   | ขั้นตอน | สิ่งที่ควรเกิดขึ้น |
   | ------- | ---------------- |
   | CRITICAL Alarm ถูกส่ง | n8n Workflow ทำงาน |
   | LINE Notify | รับข้อความแจ้งเตือน |
   | Email Notification | รับอีเมล |
   | Incident Ticket | บันทึกใน CSV/Sheets |

4. ทดสอบ Scenario AI Analysis

   | ขั้นตอน | สิ่งที่ควรเกิดขึ้น |
   | ------- | ---------------- |
   | อุณหภูมิเพิ่มเป็น 38°C | LLM วิเคราะห์ว่า Possible Cooling Failure |
   | สร้าง Alert ผ่าน MQTT | Alert ปรากฏใน Dashboard |

## บันทึกผลการทดลอง

| Scenario | ระบบที่เกี่ยวข้อง | ผลลัพธ์ | ผ่าน/ไม่ผ่าน |
| -------- | ---------------- | ------- | ----------- |
| Normal Operation | ESP32 → MQTT → Dashboard | | |
| MAJOR Alarm | ESP32 → Alarm System | | |
| CRITICAL Alarm + Auto | ESP32 → MQTT → n8n → LINE/Email | | |
| AI Fault Detection | ESP32 → Pi → LLM → Alert | | |

# การทดลองที่ 8.3 การนำเสนอระบบ

## ขั้นตอนการทดลอง

1. เตรียมการนำเสนอระบบ AIoT Telecom Station ประกอบด้วย

   - สถาปัตยกรรมระบบโดยรวม (Overall System Architecture)
   - การทำงานของ Sensor Node
   - การสื่อสารผ่าน MQTT
   - NOC Dashboard แบบ Real-time
   - Edge Gateway และ Protocol Translation
   - ระบบ Alarm และการจำแนกระดับ
   - Edge AI / LLM สำหรับ Fault Prediction
   - Workflow Automation และการแจ้งเตือน

2. สาธิตการทำงานของระบบแบบ End-to-End

   - แสดง Dashboard ขณะทำงานปกติ
   - สร้างเหตุการณ์จำลอง (จำลอง Cooling Failure)
   - แสดง Alarm ที่เกิดขึ้น
   - แสดงการวิเคราะห์ของ LLM
   - แสดงการแจ้งเตือนผ่าน LINE/Email
   - แสดง Incident Ticket ที่ถูกสร้าง

3. อภิปรายและตอบคำถาม

### คำถามท้ายการทดลอง

1. จากการทดลองทั้ง 8 Lab จงอธิบายความสัมพันธ์ระหว่างระบบต่าง ๆ ใน Telecom Base Station Remote Monitoring System
2. ถ้าต้องการนำระบบนี้ไปใช้งานจริง ควรพัฒนาเพิ่มเติมในส่วนใดบ้าง
3. จงยกตัวอย่างกรณีใช้งานจริง (Use Case) ที่ระบบนี้สามารถประยุกต์ใช้ได้

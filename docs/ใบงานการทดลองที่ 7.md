## ขั้นตอนการทดลอง

1. ติดตั้ง Node.js บน Raspberry Pi

   ```bash
   curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
   sudo apt install -y nodejs
   node -v
   npm -v
   ```

2. ติดตั้ง n8n

   ```bash
   sudo npm install n8n -g
   ```

3. รัน n8n

   ```bash
   n8n start
   ```

# การทดลองที่ 7.2 การสร้าง Workflow: Alarm → Notification

## ขั้นตอนการทดลอง

1. ใน n8n Dashboard สร้าง Workflow ใหม่ชื่อ `Alarm Notification`
2. เพิ่ม Node แรก: **MQTT Trigger**
   - Broker: `localhost:1883`
   - Topic: `site/+/alarm`
3. เพิ่ม Node ที่สอง: **Switch** (Filter ระดับ Alarm)
   - Condition 1: `{{ $json.alarm_level }}` equals `CRITICAL`
   - Condition 2: `{{ $json.alarm_level }}` equals `MAJOR`
   - Condition 3: `{{ $json.alarm_level }}` equals `MINOR`
   - Default: NORMAL (ไม่ต้องทำอะไร)
4. เพิ่ม Node ที่สาม (ต่อจาก CRITICAL): **Webhook** หรือ **Email**
   - เลือก Email node
   - กำหนด SMTP (Gmail: smtp.gmail.com, port 587)
   - กำหนด To: อีเมลของ NOC Engineer
   - Subject: `[CRITICAL] Alarm at Site {{ $json.site_id }}`
   - Body: รายละเอียด Alarm

   ![Alarm Workflow Example](images/lab-7/alarm-workflow.svg)

5. เปิดใช้งาน Workflow
6. ทดสอบโดยส่ง Critical Alarm จาก ESP32 ผ่าน MQTT

   ```cpp
   // ส่ง Alarm ผ่าน MQTT เมื่อตรวจพบเหตุการณ์ Critical
   void sendAlarm(const char* level, const char* event) {
     StaticJsonDocument<200> doc;
     doc["site_id"] = "A-01";
     doc["alarm_level"] = level;
     doc["event"] = event;
     doc["timestamp"] = time(nullptr);

     String payload;
     serializeJson(doc, payload);
     mqttClient.publish("site/a01/alarm", payload.c_str());
   }

   // เรียกเมื่อเกิด Critical
   sendAlarm("CRITICAL", "Motion + Door Open Detected");
   ```

# การทดลองที่ 7.3 การแจ้งเตือนผ่าน LINE และสร้าง Incident Ticket

## ขั้นตอนการทดลอง

1. สร้าง LINE Notify Token
   - ไปที่ https://notify-bot.line.me/my/
   - Log in ด้วยบัญชี LINE
   - คลิก Generate Token
   - ตั้งชื่อ เช่น `NOC Alarm`
   - เลือกกลุ่มหรือแชทส่วนตัว
   - คัดลอก Token

2. ใน n8n เพิ่ม Node **HTTP Request** ต่อจาก CRITICAL branch
   - Method: POST
   - URL: `https://notify-api.line.me/api/notify`
   - Header: `Authorization: Bearer <LINE_TOKEN>`
   - Body (Form-Data):
     - `message`: `[CRITICAL] Site {{ $json.site_id }}: {{ $json.event }}`

3. เพิ่ม Node **Spreadsheet File** หรือ **MySQL** สำหรับบันทึก Incident Ticket
   - สร้าง Google Sheets (หรือ CSV บน Pi)
   - Fields: Timestamp, Site ID, Alarm Level, Event, Status

4. ทดสอบระบบโดยรวม — สร้างเหตุการณ์ Critical Alarm ครบวงจร

   ```
   ESP32 Alarm → MQTT → n8n → LINE Notify + Email + Incident Log
   ```

### คำถามท้ายการทดลอง

1. Workflow Automation มีความสำคัญอย่างไรในการจัดการ Alarm ของสถานีฐานโทรคมนาคม
2. ถ้า n8n หยุดทำงาน (Down) จะมีผลต่อการแจ้งเตือนหรือไม่ และมีวิธีแก้ไขอย่างไร
3. ในการใช้งานจริง ควรแจ้งเตือน Alarm ทุกระดับทาง LINE หรือเฉพาะ Critical เท่านั้น เพราะเหตุใด
